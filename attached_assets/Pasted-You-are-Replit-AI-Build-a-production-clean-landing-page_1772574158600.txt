VALIDATION (Zod)
- name: 2–80 chars
- ranchCompany: 2–120
- town: 2–80 (dropdown list + “Other”)
- phone: simple permissive regex
- email: valid email
- crops: min 1
- acresBand: required
- role: required
- peakMonths: min 1
- commit* checkboxes must be true to submit
- oneThingToStopDoing: 0–180 chars
- honeypot must be empty

ANTI-SPAM
- Honeypot hidden input; reject if non-empty.
- Rate limit by IP (in-memory ok) on /api/apply.
- Hash IP with SHA-256 + salt (use ADMIN_PASSWORD as salt or separate salt env var if you want).

ADMIN
- /admin page: simple password prompt (client side) -> stores password in memory only (not localStorage).
- Admin requests send header: x-admin-password.
- Admin table columns: createdAt, name, ranchCompany, town, crops, acresBand, role, phone, email, ranchCount, needsMultiRanch, commit flags.
- Add CSV export button (client-side export from fetched JSON).
- Add “Copy lead” button per row.
