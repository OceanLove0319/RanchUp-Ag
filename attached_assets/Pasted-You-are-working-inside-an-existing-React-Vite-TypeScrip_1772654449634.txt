You are working inside an existing React + Vite + TypeScript frontend-only prototype called KEBB Ag™.
It uses wouter for routing, zustand for state (client/src/lib/store.ts), and localStorage for persistence.
There is NO backend. Everything must work offline.

GOAL
Add a deterministic “Packets” system that the dashboard can produce at 3 levels:
1) End-of-Day Packet (Today)
2) Monthly Packet
3) Season Packet (audit binder)
and implement a KEBB Season Packet TABLE-OF-CONTENTS (exact page order + section headers) that the PDF generator can render deterministically from existing state: Vault (logs), Blocks, Flags.

HARD RULES
- Offline-only. No API calls required.
- Deterministic outputs: same data → same packet sections/pages.
- No “AI” or random wording. Use strict templates.
- Reuse existing store models (Ranches, Blocks, Logs/Vault, Inputs Library, VarianceFlags).
- PDF can be generated via existing export utility; if none exists, add `jspdf` (preferred).
- Must be mobile-first, truck-optimized, and extremely simple to operate.

WHAT THE DASHBOARD MUST PRODUCE (3 LEVELS)
A) TODAY PACKET (“End-of-Day Proof”)
- Purpose: daily artifact a foreman can send in 15 seconds.
- Includes: today’s logs grouped by block, totals (materials + acres + est spend), issues list, PDF + share buttons.
B) MONTHLY PACKET (“Monthly Cleanup + Buyer Readiness”)
- Purpose: monthly review bundle for manager/PCA and packer QA.
- Includes: month’s logs, month totals, open issues + corrections, signoff page.
C) SEASON PACKET (“Audit Binder”)
- Purpose: packer-shipper audited season-long documentation + rollups.
- Includes: TOC, block register, field history summaries, full application records, PHI/REI summary, exceptions/corrections, rollups, exports appendix.
