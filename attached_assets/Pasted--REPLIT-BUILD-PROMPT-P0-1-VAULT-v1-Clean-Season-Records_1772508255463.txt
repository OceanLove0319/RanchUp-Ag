## Objective
Transform the Vault page into a clean, block-aware, season-aware records cabinet and add an “Export Season Packet (PDF)” experience using a print-ready route + CSS (window.print). No backend needed.

## Requirements

### 1) Vault Layout: Block + Season First
On `/vault`:
- Add a Block selector at top (chips or dropdown).
  - Default selection: last viewed block OR first block.
- Show a Season Window Badge under selector:
  - Use existing `getActiveSeasonWindow(block)` returning startISO/endISO/label.
  - Display label like: “Stone Fruit Early: Feb 1 – Jun 30 (Pacific)”
- Show three content sections (Cards, not tables):
  1) **Sprays** (FUNGICIDE/INSECTICIDE/HERBICIDE/ADJUVANT/OTHER)
  2) **Fertility** (FERTILIZER)
  3) **Notes** (if notes exist across logs, show them inline; otherwise omit)

Each section:
- Shows chronological list within active season window.
- Each row/card item shows:
  - Date (dateApplied)
  - Chemical name (chemicalName)
  - Method (method)
  - Cost (formatted USD if present)
  - Notes (trim to 1 line, expand/collapse)

### 2) Filtering Rules (must be consistent)
Vault shows ONLY logs where:
- log.blockId === selectedBlock.id

- Season Spend (sum of estimatedCost within window)

Use existing rollup utilities if present; otherwise implement small helpers.

### 4) Export to PDF (Print Packet)
Add primary button near the top:
- “Export Season Packet (PDF)”

Implementation:
- Create route: `/vault/print/:blockId`
- This route renders a print-optimized page with:

- Open `/vault/print/:blockId` in same tab and auto-trigger `window.print()` on mount.

### 6) Keep Aesthetic Consistent
On normal `/vault`, use existing dark theme cards.
