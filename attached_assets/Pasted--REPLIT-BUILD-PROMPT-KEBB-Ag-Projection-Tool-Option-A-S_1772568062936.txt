# REPLIT BUILD PROMPT — KEBB Ag “Projection Tool (Option A)” = Season Budget by Block (Plan vs Actual)

Build a clean, modern prototype module inside an existing React + TypeScript + Vite + Tailwind app (dark mode). Use Zustand for state and localStorage persistence. Use wouter for routing. DO NOT add a backend—everything is client-side.

## Goal
Add a “Projection Tool” that lets a ranch manager select a **Program Template** for each block and instantly see:
- Projected season spend (block + ranch)
- Actual-to-date spend (from logs)
- Remaining projected spend
- Over/Under vs plan
- Top cost drivers (materials)

Keep it fast, orchard-operator friendly, and not data-heavy.

- /app/projections/block/:blockId (Block Projection Detail)
- /app/projections/templates (Program Templates Manager)

## Data Model (Zustand Store + localStorage)
### Existing entities assumed
- Ranch, Block, LogEntry, MaterialLibraryItem
- LogEntry has: blockId, ranchId, type, dateTime, materialName, rateValue, rateUnit, totalMaterialNeeded, matchedLibraryItemId?, estimatedCost, flags[]

### Add new entities
1) ProgramTemplate
- id: string
- name: string (e.g., “Stone Fruit Base Program”)
- cropTags: string[] (PEACH/NECT/PLUM/CITRUS/NUTS etc.)
- notes?: string
- lines: ProgramLine[]

2) ProgramLine
- id: string
- type: "SPRAY" | "NUTRITION" | "FERTIGATION"
- materialId?: string (optional link to library item)
- materialName: string (fallback if no materialId)
- rateValue: number
- rateUnit: string (support per-acre units: gal/ac, lb/ac, qt/ac, oz/ac)
- passesPlanned: number (default 1)
- monthHint?: number (1–12) optional; if absent, treat as “unscheduled”

3) BlockProjection
- blockId: string
- templateId: string
- overrides: {
    lines?: Record<string, { rateValue?: number; passesPlanned?: number; rateUnit?: string }>
  }
- updatedAt: string (ISO)

Persist:
- templates[]
- blockProjections[] keyed by blockId
- (reuse existing localStorage patterns)

## Projection Math (deterministic + guarded)
### For each ProgramLine on a Block:
- Only compute totals if:
  - block.acres > 0
  - rateUnit contains "/ac"
- plannedTotalMaterial = block.acres * rateValue * passesPlanned

### Planned Cost
- Resolve material:
  - If materialId exists, use that library item.
  - Else attempt alias match on materialName to library (same matcher used in Quick Log).
- If no match => plannedCost = 0 and flag "UNKNOWN_MATERIAL"
- If match but base unit incompatible with rate unit => plannedCost = 0 and flag "UNIT_MISMATCH"
- If match:
  - Use midpointPrice = (priceLow + priceHigh)/2
  - Apply pack-size normalization if library item priced by pack (e.g., 2.5 GAL jug), normalize to base unit price
  - plannedCost = normalizedPlannedTotal * normalizedUnitPrice

### Actual Cost (from logs)
- actualToDate = sum(log.estimatedCost) where:
  - log.blockId matches
  - log.flags does NOT include UNIT_MISMATCH
  - log.estimatedCost > 0
- Allow filtering by date range (month/season) but default is season-to-date.

### Remaining + Over/Under
- plannedSeasonTotal = sum(plannedCost across lines)
- remaining = max(plannedSeasonTotal - actualToDate, 0)
- overUnder = actualToDate - plannedSeasonTotal

### Cost Drivers
- For a block or ranch: group plannedCost by material name and show top 5.
