You are working inside the existing KEBB Ag React + Vite frontend-only prototype.

DO NOT:
- add a backend
- break existing routes
- remove the current Guided Step Rail on Home
- make the UI feel heavy or cluttered
- introduce complicated enterprise-style workflow language

GOAL
Add a persistent guided-flow orientation system so users always know:
- where they are
- what guided step they are on
- what step comes next
- how to get back to the guided dashboard flow

The current Guided Step Rail on Home is good, but it disappears once users go deeper into the app.
Fix that by creating a lightweight persistent “direct me back” experience across key screens.

==================================================
FEATURE NAME
==================================================

Create a new global UI component:

GuidedFlowReturnBar

This component should act like:
- a persistent orientation layer
- a return-to-flow control
- a next-step prompt
- a progress reminder

==================================================
CORE UX INTENT
==================================================

This feature should answer:
- Where am I in the guided flow?
- What should I probably do next?
- How do I get back to the main guided dashboard?

This is not just navigation.
This is a lightweight guidance system.

Do NOT use a giant fixed header that eats vertical space on mobile.

==================================================
NEXT-STEP LOGIC
==================================================

Add deterministic next-step logic based on user behavior and current app state.

Track:
- currentGuidedStep
- lastMeaningfulAction
- recommendedNextStep

Examples of action-to-next-step logic:

1) After saving a log
- current = Log
- next = Review

2) After fixing a review issue
- if more review issues remain:
  next = Review
- otherwise:
  next = Plan

3) After applying a template
- next = Log

4) After generating a report
- next = Share or Today depending on context

5) If user is inside a block page and no action has been taken
- current = Today
- next = Log

6) If sparse data / brand new user
- default current = Today
- next = Log

Use practical, deterministic logic only.
No AI needed.

==================================================
STEP MAPPING BY PAGE
==================================================

Map pages to guided steps approximately like this:

- Home / Today panel -> Today
- Block detail -> Today
- Quick Log -> Log
- Review queue / issue fix -> Review
- Plan page / templates applied planning -> Plan
- Reports / exports / vault share actions -> Share
- Inputs Library -> preserve last guided step if possible, otherwise Today
- Manage pages -> preserve last guided step and show “Back to Guided Flow”
