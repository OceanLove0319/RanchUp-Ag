CATEGORIES + TERMS
1) Core Ranch Structure
- Ranch — The overall operation (one business) that contains multiple blocks.
- Block — A defined planting area (variety + rootstock + year + acres + irrigation system). The core unit for logging and costing.
- Acres (Block Acres) — The measured size of the block used for “total material needed” and cost math.
- Crop / Commodity Tag — Labels like Almond / Pistachio / Citrus / Stone Fruit used to filter templates, materials, and reports.
- Season — The time window you’re tracking (e.g., 2026 season). Logs roll up by season.

2) Logging & Execution
- Quick Log — Fast entry for what happened today (spray, fertigation, irrigation set, mowing, etc.) tied to a block.
- Log Entry — One recorded activity (date + block + activity + materials + rates + notes).
- Activity Type — The category of work: Spray / Fertigation / Irrigation / Soil Amendment / Weed / Pest / Cultural.
- Application — A spray or injection event where products were applied at a rate.
- Rate — How much product is applied per acre or per set (e.g., 15 GAL/AC).
- Total Material Needed — Auto-calculated total product for that block: rate × acres.
- Tank / Load Helper — Optional calculator: converts total needed into number of loads based on tank size.
- Copy-Forward (Log Again) — Reuse a prior log’s products/rates for speed; edit only what changed.
- Crew / Operator — Who performed the work (foreman, irrigator, custom applicator, etc.).

3) Materials & Cost Engine
- Material — Any input used: chemical, fertilizer, adjuvant, water treatment, surfactant, etc.
- Material Library — The standardized list of materials with units, pack sizes, and planning prices.
- Unit — How a material is measured (GAL, LB, OZ, TON, QT, PT).
- Rate Unit — The “per-acre” format (GAL/AC, LB/AC, OZ/AC).
- Pack Size — Container size used for pricing (e.g., 2.5 GAL, 50 LB bag).
- Normalized Unit — The base unit after converting pack size to a per-unit cost (e.g., $/GAL or $/LB).
- Planning Price Range — Low/high cost estimate used until an invoice replaces it.
- Estimated Cost — What the engine calculates for a log based on rate × acres × unit price.
- Actual Cost — Invoice/quote-entered cost that overrides planning estimates.
- Cost Engine — The math layer that converts logs into spend totals safely and consistently.

4) Templates & Programs
- Template — A pre-built “recipe” for a common job (e.g., bloom spray, hull-split, N shot, frost set).
- Template Library — Your saved templates (by crop, timing, or problem).
- Template Fill — Applying a template to a block/date to generate a ready log entry.
- Standard Program — A repeatable seasonal plan (e.g., pre-bloom → bloom → post-bloom → summer).
- Timing Tag — “Bloom,” “Petal fall,” “Hull split,” “Post-harvest,” etc. used to organize templates.

5) Projections & Variance (Money + Materials)
- Block Projection — What you expect to spend/apply on that block over a period (week/month/season).
- Season Projection — Expected total spend for the ranch or commodity for the season.
- Budget (Planned Spend) — A target spend number used to compare against actuals.
- Variance — The difference between planned and actual spend/use: actual − planned.
- Variance Flag — A warning when variance crosses a threshold (e.g., +15% over plan).
- Over/Under Indicator — Simple label showing you’re trending above or below projection.
- Spend Allocation — Splitting spend by block, commodity, activity type, or material category.

6) Flags & Data Quality
- Unit Mismatch — Entered unit/rate doesn’t match the library unit, so cost math is blocked (cost set to 0) until fixed.
- Data Mismatch — The log conflicts with saved reference data (wrong block acres, wrong material unit, missing crop tag, etc.).
- Missing Data Flag — Required fields aren’t filled (no acres, no unit price, no rate, no date).
- Outlier Flag — A value is unusually high/low compared to normal ranges (e.g., 80 GAL/AC UN-32).
- Duplicate Log Warning — Similar logs entered twice for the same block/date/activity.
- Review Queue — A list of flagged items needing correction (units, prices, acres, duplicates).
- Locked Fields — Protected values that shouldn’t change casually (block acres, base units) without admin confirmation.

7) Compliance & Reporting (Central Valley reality)
- Compliance Packet — An organized export of logs for audits, QA, and reporting (sprays, fertigation, irrigation notes).
- Audit-Ready Record — A log that has required details: date, block, products, rates, acres, operator.
- Export — Download/share reports (PDF/CSV) for PCA, packer-shipper QA, or internal records.
- Retention — How long records are kept in the vault (policy-driven).
- Pesticide Use Reporting (PUR) Support — Reports structured to help monthly reporting (not a replacement for legal requirements, just organized output).

8) App Experience / Navigation
- Active Apps — The modules currently enabled in your account (e.g., Logging, Templates, Cost Engine, Projections, Vault, Export).
- Module — A functional area of the product (Templates, Vault, Projections).
- Role — Permissions level (Owner/Admin, Foreman, Irrigator, PCA view-only, Accounting).
- Workspace — Your operation’s “home” inside the app (ranches/blocks/users/settings).
- Vault — The permanent storage area for season logs, exports, and compliance packets.
