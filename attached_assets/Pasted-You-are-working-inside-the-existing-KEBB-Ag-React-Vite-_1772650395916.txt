GOAL
Build a new MVP feature called:

“Suggested for This Block”

This feature should help growers and foremen quickly see the most relevant likely next inputs or actions for a specific block based on existing block data, recent logs, templates, inventory, timing, and issue flags.

This is NOT an “AI prescription” tool.
This IS a smart operational recommendation layer.

The experience should answer:
“What is most likely worth doing next on this block?”

==================================================
SUGGESTION ENGINE RULES (MVP)
==================================================

Build a simple deterministic rules engine.
No LLM calls. No backend inference.

Create a function like:
getSuggestionsForBlock(block, logs, templates, inventory, materials): BlockSuggestion[]

Start with practical rules like these:

RULE GROUP A: RECENCY / GAP
- If no nutrition-related input has been logged in X days, create a low/medium confidence nutrition suggestion.
- If no herbicide-related input has been logged in a long interval, create a weed-control gap suggestion.
- If no dormant spray material exists during a dormant season tag, create a dormant program gap suggestion.
- If block has issue flags mentioning mites, disease, weed escapes, deficiency, or infiltration, bias toward relevant categories.

RULE GROUP B: TEMPLATE MATCH
- If a block has an attached template, suggest the next template-linked product/material not recently logged.
- If a crop template exists and the block matches that crop, surface it as medium confidence.

RULE GROUP C: INVENTORY MATCH
- If a suggested item exists in inventory, increase confidence and show “On hand”.
- If enough quantity exists for block acres using rate x acres, show “Enough on hand”.
- If inventory is low, show “Low stock”.

RULE GROUP D: SIMILAR BLOCKS
- If 2+ similar blocks (same crop or same crop + variety) received the same input in the last 14 days, suggest that item at medium confidence.
- Show reason line:
  “Used recently on similar blocks”

RULE GROUP E: TIMING
- If the app has season tags / stage tags, use them.
Examples:
  - dormant
  - bloom
  - post-bloom
  - fruit set
  - sizing
  - hull split
  - post-harvest
- Use tags only as operational cues, not agronomic prescriptions.

CONFIDENCE LOGIC
- HIGH = 4+ strong signals
- MEDIUM = 2–3 signals
- LOW = 1 weak signal or incomplete data

==================================================
TOTALS / RATE LOGIC
==================================================

If a suggestion has:
- suggestedRate
- suggestedRateUnit containing /AC
- block.acres

Then compute:
estimatedTotal = suggestedRate * block.acres

Examples:
- 15 GAL/AC x 22.5 AC = 337.5 GAL
- 3 LB/AC x 40 AC = 120 LB

Show this as:
- Est. Total Needed: 337.5 GAL

If no acreage or no /AC rate, do not compute totals.
