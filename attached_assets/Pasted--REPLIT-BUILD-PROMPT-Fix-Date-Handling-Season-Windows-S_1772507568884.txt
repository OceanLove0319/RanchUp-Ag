## Core Problem
The prototype currently sets default dates using:
`new Date().toISOString().split('T')[0]`
This generates a UTC date string. In California (America/Los_Angeles), evening logging (e.g., 5:00 PM) can roll over to the next UTC day, causing spray logs to default to tomorrow’s date. This breaks trust and compliance accuracy.
